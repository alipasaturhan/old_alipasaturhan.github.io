Win32 API'lere Dalış: Teori



Win32 API

Geliştirilen uygulamaların işletim sistemi ile iletişimini sağlar. 32 ve 64 bit Windows işletim sistemleri için özellikle C ve C++ programlama dillerinde kullanılan uygulama geliştirme arayüzüdür.

Bir diğer tanımı ise geliştirilen uygulamalar ile Windows API kullanıldığında Windows işletim sistemi içinde .dll dosyalarda yerleşik olarak bulunan fonksiyon veri ve yapıların uygulama içinde kullanılmasını sağlar.

Sistem fonksiyonları olarak adlandırılan Windows API fonksiyonları Windows işletim sistemi ile birlikte yüklenir ve Windows sistem klasörleri altındaki .dll uzantılı dosyalarda yer alırlar. Geliştirilecek uygulamaların başına eklenen başlık dosyası ile birlikte uygulamalara dahil edilerek kullanılırlar.

windows.h

Windows API'sindeki tüm işlevler, Windows programcıları tarafından kullanılan tüm ortak makrolar ve çeşitli işlevler tarafından kullanılan tüm veri türleri için bildirimler içeren C ve C ++ programlama dilleri için Windows'a özgü bir başlık dosyasıdır. Geliştirilecek uygulamaların başına eklenen <windows.h> başlık dosyası ile birlikte uygulamalara dahil edilerek kullanılırlar.

    • Fonksiyonlarda A ve W Son Ek Arasındaki Fark

    Örneğin CreateProcess fonksiyonlarını kullanırken parametre olarak girdiler yazarız. Bu parametrelerdeki veriler ASCII (1 byte) tablosunu mu yoksa UNICODE (2 byte) tablosunu mu kullanacağını belirtmemiz lazım. Bu yüzden A ve W olarak iki adet fonksiyon ortaya çıkıyor. A: ASCII ve W: WIDE Chars yani UNICODE anlamına gelir.

    Ayrıca bazı parametrelerin veri tipleri değiştirilir. Bunun nedeni yukarıda açıklandığı gibi karakter encoding olayıdır. Örnek olarak A eki olan fonksiyonda LPCSTR, W eki olan fonksiyonda ise LPCWSTR tipi kullanılır.

    Bu düzen diğer fonksiyonlar için de geçerlidir (CreateProcessAsUser gibi). Fakat her fonksiyonda ASCII desteklenmez. Bu yüzden sadece UNICODE tablosunu kullanarak fonksiyonları kullanmamız gerekir. Örneğin; CreateProcessWithLogonW ve CreateProcessWithTokenW gibi.

    processthreadsapi.h başlığı altında CreateProcess fonksiyonunu yalın halde kullandığınız taktirde hangi fonksiyonun kullanılacağına karar veren compiler emir komutlarını görüyorsunuz: 

    #ifdef UNICODE
#define CreateProcess  CreateProcessW
#else
#define CreateProcess  CreateProcessA
#endif // !UNICODE

    Fonksiyon Parametreleri

  • CreateProcess

    • BOOL CreateProcessA(
  [in, optional]      LPCSTR                lpApplicationName,
  [in, out, optional] LPSTR                 lpCommandLine,
  [in, optional]      LPSECURITY_ATTRIBUTES lpProcessAttributes,
  [in, optional]      LPSECURITY_ATTRIBUTES lpThreadAttributes,
  [in]                BOOL                  bInheritHandles,
  [in]                DWORD                 dwCreationFlags,
  [in, optional]      LPVOID                lpEnvironment,
  [in, optional]      LPCSTR                lpCurrentDirectory,
  [in]                LPSTARTUPINFOA        lpStartupInfo,
  [out]               LPPROCESS_INFORMATION lpProcessInformation
);

  • lpApplicationName Çalıştırılabilir PE dosyanın path değerini alır. lpApplicationName parametresi NULL olabilir. Bu durumda, modül adı, lpCommandLine dizesindeki ilk boşlukla ayrılmış belirteç olmalıdır. Bu yöntem 16 bitlik çalıştırılabilir dosyalar için zorunludur fakat tüm uygulamalarda bu yapılabilir. Bu durumda 16 bitlik ise lpCommandLine tarafından gösterilen dize, yürütülebilir dosyayı ve parametrelerini belirtmelidir.
  • lpCommandLine Çalıştırılabilir PE dosyanın alabileceği gerekli veya isteğe bağlı parametrelerini alır. Örnek olarak lpApplicationName parametresine notepad.exe verildiği zaman bir .txt dosyası ismi vermek gibidir. Böylece sadece notepad.exe değil verilen .txt dosyası açılır. MAX_PATH, sonlandırma karakteri de (0x00) dahil 2^15-1 uzunluktadır. Bu işlevin Unicode sürümü CreateProcessW, bu dizenin içeriğini değiştirebilir. Bu nedenle bu parametre read-only belleğe (bellek hücresine) bir işaretçi olamaz (örneğin const bir değişkene). Eğer olursa fonksiyonda erişim ihlallerine neden olabilir. Ayrıca bu parametre NULL ise commandLine olarak lpApplicationName tarafından gösterilen dize kullanılır. İşletim sistemi her iki parametreyi birleştirmek için (lpApplicationName ve lpCommandLine) GetCommandLine fonksiyonunu çağırır (komutun işlevine bakmak için ilgili başlığa gidiniz). lpApplicationName NULL ise komut satırının boşlukla ayrılmış ilk belirteci modül adını (yani lpApplicationName yerine) belirtir.
  • lpProcessAttributes Oluşturulacak yeni process için isteğe bağlı güvenlik ayarlarını ayarlayan parametredir. Sub process'ler (child process) için devralınıp devralınamayacağını belirtir. Oluşturulacak process için güvenlik ayarlarını barındıran bu parametre, Windows sistemdeki bir C++ structure yapısının pointer'ını tutar. Pointer'ın tuttuğu bu yapıyı güvenlik ayarlarını barındıran bir küme olarak düşünebiliriz. Bu yapının ismi SECURITY_ATTRIBUTES idir. lpProcessAttributes NULL ise process devralınamaz. Eğer lpProcessAttributes veya SECURITY_ATTRIBUTES.lpSecurityDescriptor NULL ise, oluşturulacak process default değerlerde SECURITY_DESCRIPTOR oluşturacak. İlgili başlığa giderek bu yapı hakkında bilgi alabilirsiniz.
  • lpThreadAttributes Oluşturulacak yeni thread için isteğe bağlı güvenlik ayarlarını belirtir. Tıpkı lpProcessAttributes mantığındadır fakat burada her şey thread için geçerlidir. Örneğin parametre olarak lpThreadAttributes veya lpSecurityDescriptor NULL ise oluşturulan thread için güvenlik ayarları default security descriptor kullanır.
  • bInheritHandles
  • dwCreationFlags Process oluşumunu etkileyen çeşitli flag'leri vardır. Bunların hepsini görmek için: MSDN -> Procss Creation Flags
  • lpEnvironment Yeni process için environment bloğunu tutan pointer. Bu parametre NULL ise yeni process çağıran process'in ortamını kullanılır. Peki nedir bu environment bloğu?
  • lpCurrentDirectory İsteğe bağlı olarak ayarlanan bu parametre yeni oluşturulacak olan process'in working directory path'idir. Dize (string) ayrıca bir UNC (Universal Naming Convention) yolu belirtebilir. Bu parametre NULL ise yeni process çağıran işlemle aynı geçerli sürücüye ve dizine sahip olacaktır.
  • lpStartupInfo STARTUPINFO veya STARTUPINFOEX yapısının işaretini tutar. Gelişmiş özelliklerle ayarlamak istenirse STARTUPINFOEX kullanılır. Eğer STARTUPINFOEX kullanılacaksa dwCreationFlags parametresinde EXTENDED_STARTUPINFO_PRESENT belirtilmeli.
  • lpProcessInformation Yeni process hakkında id bilgilerini alan bir PROCESS_INFORMATION yapısının işaretini alır. PROCESS_INFORMATION hakkında daha fazla bilgi için ilgili başlığa gidiniz.

  • SECURITY_ATTRIBUTES (Windows)

    • Güvenlik tanımlayıcısını içerir ve bu yapı belirtilerek alınan tanıtıcının devralınabilir olup olmadığını belirtir.

    Bu yapı, CreateFile, CreatePipe, CreateProcess, RegCreateKeyEx veya RegSaveKeyEx gibi çeşitli işlevler tarafından oluşturulan nesneler için güvenlik ayarları sağlar. 

    typedef struct _SECURITY_ATTRIBUTES {
  DWORD nLength;
  LPVOID lpSecurityDescriptor;
  BOOL bInheritHandle;
} SECURITY_ATTRIBUTES, *PSECURITY_ATTRIBUTES, *LPSECURITY_ATTRIBUTES;

    nLength SECURITY_ATTRIBUTES yapısının bayt cinsinden boyutunu alır. Bu parametreye değer olarak sizeof(SECURITY_ATTRIBUTES) komutu gelebilir.

    lpSecurityDescriptor Yeni bir process oluşturulduğunda döndürülen tanıtıcının devralınıp alınmayacağını belirten bir Boole değerini alır. Bu üye TRUE ise, yeni süreç tanıtıcıyı devralır.
